# gitwrite

**Private, versioned writing for people who live in the terminal.**

gitwrite watches your Markdown files and quietly commits and pushes them to your Git repository — no manual `git add`, no `git commit`, no `git push`. You write. gitwrite handles the rest.

It is built for engineers writing internal specs, researchers building private knowledge bases, AI practitioners documenting models and experiments, and anyone who wants the safety of version control without the overhead of Git discipline. Your files stay in your own repository, on your own terms — not in someone else's cloud.

---

## How it works

gitwrite runs as a background daemon. When you save a file, it waits a few seconds and then commits it locally. Every few minutes, it pushes everything to your remote. If your laptop sleeps, it catches up automatically when you wake it.

The result feels like writing in Google Docs — your work is always saved — except it is plain Markdown, it is yours, and it is fully versioned.

## Install

**npm**
```sh
npm install -g gitwrite
```

**Yarn**
```sh
yarn global add gitwrite
```

---

## Quick start

```sh
cd ~/notes
gitwrite init    # first run sets up global defaults + registers this directory
gitwrite start   # start watching
```

That is it. Write something, save it, and gitwrite commits it.

---

## Commands

| Command | What it does |
|---|---|
| `gitwrite init` | Set up gitwrite in the current directory |
| `gitwrite start` | Start watching your registered directories |
| `gitwrite stop` | Stop the daemon gracefully |
| `gitwrite restart` | Stop and restart (useful after config changes) |
| `gitwrite status` | Show what gitwrite is watching and what it has done lately |
| `gitwrite logs` | Tail the activity log |
| `gitwrite config` | Edit global defaults or local directory overrides |
| `gitwrite add [path]` | Add a directory to your watch list |
| `gitwrite remove [path]` | Remove a directory from your watch list |
| `gitwrite help` | List all available commands |

## Configuration

gitwrite stores your global config at `~/.gitwriterc.json`. Run `gitwrite config` to edit it interactively.

**Global defaults:**

```json
{
  "fileTypes":    [".md", ".mdx"],
  "debounce":     3000,
  "pushInterval": 300000,
  "branchMode":   "current",
  "watch":        []
}
```

**Per-directory overrides:**

Run `gitwrite config` from inside a watched directory to set local overrides. Overrides are stored in `.gitwrite.json` in that directory. Anything not overridden falls through to your global defaults.

---

## Branch modes

gitwrite supports two branch modes, configurable globally or per directory.

**`current`** — Commits to whatever branch you currently have checked out. Switch branches freely; gitwrite follows you.

**`autogenerated`** — Creates a fresh named branch at the start of each session. Branch names are generated by combining a random adjective, noun, and verb — like `crimson-walrus-stumbling` or `teal-fox-bouncing`. 64,000 possible combinations. Good for keeping autosave history separate from your main branch.

## Requirements

- Node.js ≥ 18
- Git
- A configured Git remote (`git remote add origin <url>`) — gitwrite requires one before it will initialize a directory

---

## Why not Google Docs, Notion, or Obsidian Sync?

Those tools store your files on their infrastructure. gitwrite does not. Your writing lives in a plain Git repository that you control, pushed wherever you want — GitHub, GitLab, a private server, anywhere that accepts a Git remote.

This is especially useful for proprietary documentation — internal specs, research notes, model cards, experiment logs — where putting files into a third-party cloud is not an option.
